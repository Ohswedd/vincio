# Reference: API

## `vincio.ContextApp`

```python
ContextApp(name, *, objective=None, output_schema=None, config=None,
           provider=None, model=None, budget=None, policies=None, prompt_spec=None)
```

The surface is also grouped into six lazily-constructed **capability
facades** — `app.runs` (run/stream/batch/evaluate), `app.knowledge`
(sources/memory), `app.governance` (residency/cards/lineage/EU AI Act),
`app.optimization` (cost/evals/rotation/self-improvement), `app.serving`
(MCP/A2A/realtime), and `app.training` (capture/export/distill) — each a narrow,
isolated view that delegates to the same implementation. The flat `app.<method>`
API below remains fully supported.

| Method | Purpose |
|---|---|
| `configure(objective=, role=, rules=, examples=, citation_policy=, ...)` | declarative prompt setup |
| `add_source(name, path=, documents=, connector=, chunking=, retrieval=)` | load + chunk + index a knowledge source (local files, in-memory documents, or any connector) |
| `add_memory(scope=, strategy=, store=, embedder=)` | enable the memory engine (hybrid recall by default) |
| `remember(content, user_id=, agent_id=, session_id=, ...)` | ergonomic policy-checked memory write |
| `recall(query, user_id=, agent_id=, session_id=, top_k=)` | scored hybrid memory recall |
| `add_tool(fn_or_name, permissions=, approval_required=, side_effects=)` | enable a tool |
| `add_evaluator(name_or_fn)` | score every run with a metric |
| `add_validator(name, fn, blocking=)` | semantic output validator |
| `add_optimizer(name)` | register an optimization dimension |
| `set_policy(name, value)` | set a run policy (e.g. `answer_only_from_sources`) |
| `add_rail(name=, kind=, direction=, action=, ...)` | programmable input/output rail (topic / format / safety / custom) |
| `register_rail_predicate(name, fn)` | register a custom rail predicate `(text, params) -> falsy \| message` |
| `add_output_schema(schema, keywords=, task_types=, when=, priority=)` | register an alternative output schema, routed per run |
| `enable_self_correction(max_cycles=, max_cost_usd=)` | bounded validate→critique→repair on failed outputs |
| `predictor(signature, model=, temperature=)` | `Predict` bound to the app's provider for a typed `Signature` |
| `run(input, files=, tenant_id=, user_id=, session_id=, config=)` / `arun` | execute the 17-step pipeline → `RunResult`; `arun` / `astream` also accept `feature=` as a cost-attribution dimension |
| `astream(input, ...)` / `stream` | the same pipeline with end-to-end streaming → `RunStreamEvent` iterator |
| `submit(input, ...)` | start a run in the background → `RunHandle` (`cancel()` propagates cooperative cancellation; `await handle` / `result()` → `RunResult`; the cancelled run is still fully recorded) |
| `use_semantic_context_scoring(enabled=True, *, mmr_lambda=None)` | score/select context by embedding cosine + MMR with reranker-blended relevance (needs a real `retrieval.embedder`) → `ContextApp` |
| `agent(tools=, planner=, max_steps=, evaluator=)` | bounded agent handle |
| `crew(name, members=, process=, tools=, max_rounds=)` | multi-agent `Crew` over a shared blackboard (sequential / parallel / hierarchical) |
| `graph(name, state_schema=, reducers=)` | durable `StateGraph` checkpointed in the app's metadata store |
| `workflow(name)` | deterministic `Workflow` builder (approval gates pause; `resume(result, approvals=)` continues) |
| `evaluate(dataset, metrics=, concurrency=, gates=, judges=)` | `EvalReport` |
| `add_online_evaluator(metric, sample_rate=, name=)` | score a sampled fraction of live runs off the hot path; writes a score time series to the store |
| `add_metric_rail(metric, threshold=, direction=, action=, name=, **params)` | use an eval metric as a runtime guardrail |
| `experiment(name, variants=, dataset=, metrics=)` | production A/B over prompt/model/config variants → `Experiment` (`compare`/`cost`/`significance`) |
| `aflush_online()` | await in-flight online evaluations (tests/shutdown) |
| `improvement_loop(metrics=, weights=, gates=, experiment=, optimizer="evolution"\|"reflective", strategy=, ...)` | `ImprovementLoop`: trace → dataset → eval → optimize → promote on this app |
| `self_improvement(policy=, *, dataset=, golden=, registry=, prompt_name=)` | the unified self-improvement contract → `SelfImprovementController` (`astream`/`step`/`run` emit observe → proposal → meta → label → reeval → canary → promote/rollback) |
| `deploy(candidate, *, dataset=None, live_inputs=None, score_fn=None, canary=, metrics=, gates=, tag="production", registry=, rollback_on_fail=True, dry_run=False)` | canary-gated prompt/policy promotion → `DeployResult`. Offline mode (`dataset=`) gates on a held-out comparison; live mode (`live_inputs=` + `score_fn=`) ramps `CanarySpec.percent` of real runs onto the candidate via `LiveCanary` and auto-rolls-back a regression. Promotes only on a no-regression verdict |
| `use_consent_ledger(ledger=None, *, default_allow=False)` | attach a `ConsentLedger` (GDPR purpose/lawful basis), consulted by access decisions + memory recall → the ledger |
| `reflective_optimize(dataset, *, strategy="reflective"\|"mipro", metrics=, budget=12, minibatch_size=8, seed=7, weights=, gates=, objectives=, apply=False)` | GEPA-style reflective optimization → `ReflectiveResult` (drop-in `OptimizationResult` with `.frontier` / `.reflections`) |
| `enable_training_capture(enabled=True)` | record full output + cited evidence on each trace (incl. streaming) for faithful trace-based distillation → `ContextApp` |
| `export_training_set(*, name=, runs=, traces=, limit=500, min_feedback_score=, require_grounding=True, min_support=0.5, max_examples=, path=, format="openai"\|"anthropic")` | curate grounded fine-tuning JSONL from `RunResult`s (`runs=`, faithful, flag-free) or captured traces → `TrainingSet` |
| `distill(training_set, dataset, *, teacher, student, trainer=, quality_metric=, min_quality_ratio=0.97, gates=, apply=True)` | gated teacher→student distillation; promotes a `ModelCascade` on quality-hold → `DistillationResult` |
| `use_learned_compression(compressor=None)` | install a learned compressor (`LLMLinguaCompressor`) on the compiler, ungated → `ContextApp` |
| `gate_compression(dataset, *, compressor=, metrics=, min_faithfulness=0.9, min_quality_ratio=0.98)` | adopt a learned compressor only if it preserves cited facts + quality under eval → `CompressionTuningResult` |
| `calibrate_judge(judge, samples, *, budget=4)` | reflectively tune a `GEvalJudge`'s steps for Cohen's-κ agreement → `JudgeCalibrationResult` |
| `use_learned_budgets(source)` | install eval-tuned per-task budget allocations (`LearnedAllocations`, path, or mapping) |
| `use_pack(pack, set_schema=, merge_rules=)` | apply a domain or vertical pack (a name or `Pack`): prompt config + schema + policies + evaluators + rails — and, for a vertical (`"healthcare"`/`"ediscovery"`/`"kyc"`/`"customer_support"`/`"code_review"`), retrieval knobs + scoped memory + a residency posture too |
| `assistant(*, user_id=, tenant_id=, session_id=, memory_writeback=True, auto_approve=, on_approval=, feature="assistant")` | open a conversational, session-aware `Assistant`: turns threaded under one session, multi-turn state via session-scoped memory write-back, and an approval surface for write tools |
| `voice_agent(*, backend="inprocess"\|"openai"\|"gemini", config=, research=True, memory_os=True, rails=True, owner_id="voice", **backend_kwargs)` | open an end-to-end `VoiceAgent` — a realtime session wired to the deep-research agent (in-session `research` tool), the memory OS, and the app's input/output rails (`vincio[realtime]` for hosted backends) |
| `add_skill(path_or_skill, register_scripts=)` | load an Agent Skill (`SKILL.md`) with progressive disclosure; optionally expose bundled scripts as sandboxed tools |
| `add_mcp_server(name, command=/url=/server=/transport=, tools=, resources=, prompts=, permissions=, sampling=, elicitation=, auth=, headers=, http_client=)` | connect an MCP server; its tools register through the permissioned runtime, resources become evidence |
| `add_mcp_from_registry(name, registry=, directory=, allow=, deny=, server=, transport=, ...)` | MCP-server marketplace bridge — discover from a registry, govern reachability (allow-list + audited `agent_resolve`), and land the server's tools in the permissioned runtime, in one call |
| `serve_mcp(name=, expose_resources=, expose_prompts=, token_validator=)` | expose this app as an MCP server (`MCPServer`) |
| `serve_a2a(target=, name=, url=, description=, token_validator=)` | expose a crew / compiled graph / the app over A2A (`A2AServer`, Agent Card + task lifecycle) |
| `batch(inputs, *, backend=, config=, discount=0.5, timeout_s=)` / `abatch` | run many inputs through a batch backend (~50% cost) → `list[RunResult]` |
| `use_cascade(models=, *, rungs=, min_confidence=0.5, max_escalations=, confidence=)` | route runs up a cheap→capable model cascade, escalating on low confidence → `ContextApp` |
| `use_router(models, *, strategy="cheapest"\|"fastest"\|"least_busy", budget_usd=, guard_capabilities=True)` | route each run to the cheapest/fastest/least-busy *capable* model, downgrading to a per-request budget → `ContextApp` |
| `gate_swap(candidate_model, *, baseline_model=, dataset=, traces=, gates=, alpha=0.05, repeats=1, flake_quarantine=True, pin_tools=True)` / `agate_swap` | gate a model swap on replayed golden traces + an eval/cost/latency/behavioral diff with significance → `SwapVerdict` |
| `swap_regression(dataset, *, candidate_model, baseline_model=, metrics=, quality_metric="lexical_overlap", alpha=0.05, repeats=1, flake_quarantine=True)` / `aswap_regression` | swap only the model on a fixed dataset → `SwapRegressionReport` (significance, cost/latency, worst slices) |
| `shadow(candidate_model, *, candidate_provider=, block=False)` | dual-dispatch a candidate for offline diff while serving the primary → `ShadowProvider` |
| `canary(candidate_model, *, percent=5.0, candidate_provider=, score_fn=, min_samples=20, regression_threshold=0.05, prompt_name=)` | ramp live traffic onto a candidate with online scoring + auto-rollback → `CanaryRouter` |
| `watch_lifecycle(models=, *, as_of=, warn_within_days=90, propose=True)` | scan pinned models for sunset and propose migrations → `{"alerts", "proposals"}` |
| `set_cost_budget(*, limit_usd, scope="tenant"\|"feature"\|"user"\|"global", id=, period="run"\|"hour"\|"day"\|"month"\|"total", on_breach="cap"\|"degrade"\|"queue_to_batch", degrade_model=, anomaly_factor=)` | enforce a spend budget with on-breach action → `ContextApp` |
| `cost_report(*, by="tenant"\|"feature"\|"user"\|"model"\|"provider"\|"run", since=)` | aggregate ledger spend along a dimension → `CostReport` |
| `enable_prompt_caching(*, ttl="5m"\|"1h", min_prefix_tokens=1024)` | mark long stable prefixes for provider prompt caching → `ContextApp` |
| `use_bandit_router(models, *, bandit="epsilon_greedy"\|"ucb1"\|"linucb", safe_model=, regret_budget=2.0)` | route live traffic through a guarded online bandit (safety floor + auto-rollback) → `ContextApp` |
| `research(question, *, objective="", budget=ResearchBudget(...), strategies=, judge=)` / `aresearch` | deep-research loop (search→read→reflect→verify→synthesize) → `ResearchReport` (cited, grounded, budgeted) |
| `enable_memory_os(*, scope="agent", owner_id=, max_core_tokens=2000, permission="memory:write")` | self-editing memory as permissioned, audited tools + a context-pressure pager → `MemoryOS` |
| `enable_computer_use(backend="mock"\|"playwright"\|"provider", *, isolation=, require_isolation=False, approval_required=True)` | register navigate/click/type/screenshot as audited, approval-gated tools → backend |
| `use_hosted_tools(names=, *, namespace="openai")` | surface provider-native hosted tools (`web_search`/`file_search`/`code_interpreter`/`computer_use`) as namespaced, permissioned tools → `ContextApp` |
| `reflective_optimize(dataset, *, reflector="heuristic"\|"llm", ...)` | GEPA-style optimization; `reflector="llm"` wires the provider-backed reflector → `ReflectiveResult` |
| `task` (decorator) | configure from a task class |
| `stats()` | sources, tools, memory, cost, run counts |

## `RunResult`

`output` (typed when a schema is set), `raw_text`, `status`, `error`,
`trace_id`, `context_packet_id`, `evidence`, `citations`, `tool_results`,
`usage`, `cost_usd`, `latency_ms`, `validation`, `eval_scores`,
`excluded_context`.

## `RunStreamEvent`

Yielded by `astream` / `stream`. `type` is one of `stage`, `text_delta`,
`partial_output` (incremental partial-JSON parse for structured output,
with `output_complete`, plus streaming validation: `valid_prefix` and
`validation_errors`), `tool_call`, `tool_result`, `usage`, `error`, or
the terminal `done` (carrying `result: RunResult`).

## Stability & versioning

From 1.0, Vincio follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html)
on its public API — every symbol re-exported from the top-level `vincio` package
(`vincio.__all__`, also returned by `vincio.stability.public_api()`) plus the
documented entry points above. See the [stability policy](stability.md) for the
deprecation contract; `vincio.deprecated` / `vincio.experimental` mark symbols
and `vincio.stability_of(obj)` introspects any symbol's guarantee.
