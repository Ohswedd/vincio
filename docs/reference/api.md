# Reference: API

## `vincio.ContextApp`

```python
ContextApp(name, *, objective=None, output_schema=None, config=None,
           provider=None, model=None, budget=None, policies=None, prompt_spec=None)
```

The surface is also grouped into six lazily-constructed **capability
facades**, `app.runs` (run/stream/batch/evaluate), `app.knowledge`
(sources/memory), `app.governance` (residency/cards/lineage/EU AI Act),
`app.optimization` (cost/evals/rotation/self-improvement), `app.serving`
(MCP/A2A/realtime), and `app.training` (capture/export/distill), each a narrow,
isolated view that delegates to the same implementation. The flat `app.<method>`
API below remains fully supported.

| `add_source(name, path=, documents=, connector=, chunking=, retrieval=)` | load + chunk + index a knowledge source (local files, in-memory documents, or any connector) |

| `add_validator(name, fn, blocking=)` | semantic output validator |

| `register_rail_predicate(name, fn)` | register a custom rail predicate `(text, params) -> falsy \| message` |
| `add_output_schema(schema, keywords=, task_types=, when=, priority=)` | register an alternative output schema, routed per run |

| `submit(input, ...)` | start a run in the background → `RunHandle` (`cancel()` propagates cooperative cancellation; `await handle` / `result()` → `RunResult`; the cancelled run is still fully recorded) |
| `use_semantic_context_scoring(enabled=True, *, mmr_lambda=None)` | score/select context by embedding cosine + MMR with reranker-blended relevance (needs a real `retrieval.embedder`) → `ContextApp` |

| `experiment(name, variants=, dataset=, metrics=)` | production A/B over prompt/model/config variants → `Experiment` (`compare`/`cost`/`significance`) |
| `aflush_online()` | await in-flight online evaluations (tests/shutdown) |
| `improvement_loop(metrics=, weights=, gates=, experiment=, optimizer="evolution"\|"reflective", strategy=, ...)` | `ImprovementLoop`: trace → dataset → eval → optimize → promote on this app |
| `self_improvement(policy=, *, dataset=, golden=, registry=, prompt_name=)` | the unified self-improvement contract → `SelfImprovementController` (`astream`/`step`/`run` emit observe → proposal → meta → label → reeval → canary → promote/rollback) |
| `deploy(candidate, *, dataset=None, live_inputs=None, score_fn=None, canary=, metrics=, gates=, tag="production", registry=, rollback_on_fail=True, dry_run=False)` | canary-gated prompt/policy promotion → `DeployResult`. Offline mode (`dataset=`) gates on a held-out comparison; live mode (`live_inputs=` + `score_fn=`) ramps `CanarySpec.percent` of real runs onto the candidate via `LiveCanary` and auto-rolls-back a regression. Promotes only on a no-regression verdict |
| `use_consent_ledger(ledger=None, *, default_allow=False)` | attach a `ConsentLedger` (GDPR purpose/lawful basis), consulted by access decisions + memory recall → the ledger |
| `reflective_optimize(dataset, *, strategy="reflective"\|"mipro", metrics=, budget=12, minibatch_size=8, seed=7, weights=, gates=, objectives=, apply=False)` | GEPA-style reflective optimization → `ReflectiveResult` (drop-in `OptimizationResult` with `.frontier` / `.reflections`) |
| `enable_training_capture(enabled=True)` | record full output + cited evidence on each trace (incl. streaming) for faithful trace-based distillation → `ContextApp` |
| `export_training_set(*, name=, runs=, traces=, limit=500, min_feedback_score=, require_grounding=True, min_support=0.5, max_examples=, path=, format="openai"\|"anthropic")` | curate grounded fine-tuning JSONL from `RunResult`s (`runs=`, faithful, flag-free) or captured traces → `TrainingSet` |

| `use_pack(pack, set_schema=, merge_rules=)` | apply a domain or vertical pack (a name or `Pack`): prompt config + schema + policies + evaluators + rails, and, for a vertical (`"healthcare"`/`"ediscovery"`/`"kyc"`/`"customer_support"`/`"code_review"`), retrieval knobs + scoped memory + a residency posture too |
| `assistant(*, user_id=, tenant_id=, session_id=, memory_writeback=True, auto_approve=, on_approval=, feature="assistant")` | open a conversational, session-aware `Assistant`: turns threaded under one session, multi-turn state via session-scoped memory write-back, and an approval surface for write tools |
| `voice_agent(*, backend="inprocess"\|"openai"\|"gemini", config=, research=True, memory_os=True, rails=True, owner_id="voice", **backend_kwargs)` | open an end-to-end `VoiceAgent`, a realtime session wired to the deep-research agent (in-session `research` tool), the memory OS, and the app's input/output rails (`vincio[realtime]` for hosted backends) |
| `add_skill(path_or_skill, register_scripts=)` | load an Agent Skill (`SKILL.md`) with progressive disclosure; optionally expose bundled scripts as sandboxed tools |
| `add_mcp_server(name, command=/url=/server=/transport=, tools=, resources=, prompts=, permissions=, sampling=, elicitation=, auth=, headers=, http_client=)` | connect an MCP server; its tools register through the permissioned runtime, resources become evidence |

## Ergonomic front door (`vincio.tasks`)

One-line, task-shaped constructors over `ContextApp`, also re-exported at the top
level and tagged `@experimental`. Each is a purely-compositional facade that
configures an app with the *same* public builder calls and **lowers to the exact
same governed `ContextApp.run` packet** as the verbose form; `facade.app` is the
escape hatch to every deep method.

| Entry point | Returns | What it composes |
|---|---|---|
| `rag(sources=None, *, evaluators=("groundedness","citation_accuracy"), grounded=True, chunking=None, retrieval="hybrid", ...)` | `RagTask` (`.ask(q)`) | `add_source` + `set_policy("answer_only_from_sources", True)` + `add_evaluator(...)` |
| `extractor(schema, ...)` | `Extractor` (`.extract(text)`) | `ContextApp(output_schema=schema)` |
| `tool_agent(tools=(), *, writes=(), approve=(), ...)` | `ToolAgent` (`.run(task)`) | `add_tool(...)` with write tools approval-gated |
| `evaluation(dataset=None, *, metrics=(), gates=None, ...)` | `Evaluation` (`.run()`) | `add_evaluator(...)` + `evaluate(dataset, gates=)` |
| `chat(*, tools=(), writes=(), approve=(), ...)` | `Assistant` | `add_tool(...)` + `app.assistant(...)` |
| `Flow(*, provider=, model=, ...)` | `Flow` | fluent, immutable `retrieve` → `ground` → `call` → `validate` → `evaluate`; `.run(input)` |

All constructors accept `provider=` / `model=` / `name=`, a `role` / `objective` /
`rules` persona, and `app=` to layer the task onto a pre-built `ContextApp`. The
byte-identical lowering is proven by `vincio.testing.run_signature` and gated by the
**ErgonomicsBench** family. See the [ergonomic-surface concept](../concepts/ergonomic-surface.md).

## `RunResult`

`output` (typed when a schema is set), `raw_text`, `status`, `error`,
`trace_id`, `context_packet_id`, `evidence`, `citations`, `tool_results`,
`usage`, `cost_usd`, `latency_ms`, `validation`, `eval_scores`,
`excluded_context`.

## `RunStreamEvent`

Yielded by `astream` / `stream`. `type` is one of `stage`, `text_delta`,
`partial_output` (incremental partial-JSON parse for structured output,
with `output_complete`, plus streaming validation: `valid_prefix` and
`validation_errors`), `tool_call`, `tool_result`, `usage`, `error`, or
the terminal `done` (carrying `result: RunResult`).

per visit, a `GatheredReputation` result), a denied peer skipped, a forged artifact refused, a gossiped revocation excluding the withdrawn claim, every peer and artifact audited. **Transitive trust & Sybil resistance:** `build_trust_model` / `TrustConfig` / `TrustModel` / `IssuerTrust` build a bounded, transitive web-of-trust rooted in the importer's local `ReputationLedger`, and (opt-in via `combine_attestations(..., trust=, trust_config=)` / `app.import_reputation` / `app.gather_reputation`) scale each issuer's contributed evidence mass by the importer's own trust in it, an issuer known first-hand at its earned weight, an issuer a trusted one vouches for at most a hop away with decay, an unknown one floored never zeroed, so a mutually-vouching Sybil cluster cannot outvote a few admitting a thin or low-trust standing on conservative terms rather than refusing it, the ceiling ramping toward parity as settled history accrues and a regression walking it back; `verify()` re-derives the terms from the bound standing so a tampered ceiling is caught even after re-sealing, and `bound_position` / `apply_to_terms` fold it into the negotiation/contracting path. **Collateralized escrow:** `Escrow` / `EscrowConfig` / `EscrowVerification` / `post_escrow` / `settle_escrow` bind the admission-required collateral to a specific `Contract` into a signed, offline-verifiable artifact (the held amount re-deriving from the admission posture); settling the contract releases the whole stake on a fulfilled delivery and forfeits a bounded slice proportional forfeiture is caught even after re-sealing. ## Stability & versioning

From 1.0, Vincio follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html)
on its public API, every symbol re-exported from the top-level `vincio` package
(`vincio.__all__`, also returned by `vincio.stability.public_api()`) plus the
documented entry points above. See the [stability policy](stability.md) for the
deprecation contract; `vincio.deprecated` / `vincio.experimental` mark symbols
and `vincio.stability_of(obj)` introspects any symbol's guarantee.
