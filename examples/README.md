# Vincio examples

All examples run **fully offline** by default using the deterministic mock
provider — no API keys needed. To run against a real model:

```bash
export VINCIO_PROVIDER=openai VINCIO_MODEL=gpt-5.2-mini OPENAI_API_KEY=sk-...
```

| `55_world_model_planning.py` | world-model / simulation-based planning — a `WorldModel` fit offline from recorded reset/step `Transition`s that learns each tool's parameterized effect under a learned precondition (and generalizes over arguments), a `CalibrationReport` that earns the model planning weight only once its predictions track the real environment, and a `ModelPredictivePlanner` that searches imagined rollouts with the test-time-search beam and beats a reactive one-step planner at a fixed action budget (`record_transitions` / `WorldModel.predict` / `WorldModel.imagine` / `ModelPredictivePlanner.plan`) |
| `56_record_replay_debugger.py` | causal record-replay debugger — a `Recorder` that captures every non-deterministic edge of a run (model responses, tool outputs, retrieval hits, negotiated capabilities, clock/seed) into a portable, content-addressed `Recording`, a `Replayer` that re-runs it byte-for-byte (the recording, not the live provider, serves the answer) and reports a `Divergence` when the code under replay has changed, and branch-and-edit (`BranchEdit`) that forks the recording and re-executes only the affected suffix while the unchanged prefix is still served from the recording |
| `57_learned_semantic_cache.py` | learned semantic cache & near-miss KV reuse — a `LearnedSemanticCache` that answers a *semantically-equivalent* (not byte-identical) request from cache, with a `ThresholdCalibrator` that fits the acceptance threshold from labelled trace pairs so an accepted near-miss clears a precision target (never serving below the bar), a `SemanticCacheGate` that catches a drifted cache with the same eval-replay regression check that gates a model swap, and a `KVPrefixPool` that reuses a shared stable-prefix KV footprint across a request family — all held under the resident-memory budget (`app.use_semantic_cache` / `app.use_kv_prefix_reuse` / `app.semantic_cache_report` / `app.kv_prefix_report`) |

| `58_on_device_local_adaptation.py` | on-device fine-tuning & continual local adaptation — a `LocalLoRATrainer` that fits a parameter-efficient, low-rank `LocalAdapter` in-process from the flywheel's grounded data, an `AdaptedProvider` that applies it to the in-process model so in-distribution traffic is answered the way it was taught while off-distribution traffic falls through untouched (bounded), an `AdapterGate` that promotes a new version only when the adapted model is at-least-as-good as its base (the same no-regression discipline a hosted fine-tune job clears), and an `AdapterRegistry` that versions adapters and rolls them back — every step in-process, gated, and reversible (`app.adapt_locally` / `app.local_adaptation` / `app.use_local_adapter`) |
| `59_federated_cross_org_self_improvement.py` | federated / cross-org self-improvement — each member builds a numeric, raw-text-free `Contribution` (the clipped, masked subspace scatter of its local adapter geometry, never a prompt or response) behind the consent ledger and residency posture, a `SecureAggregator` merges the fleet's contributions into a shared `FederatedSubspace` (the masks cancel exactly, so no single member's update is ever seen) and refuses a round below the k-anonymity contributor floor, and the adopting member re-fits its *own* adapter against the shared geometry and adopts it only when at-least-as-good as its base — the same no-regression gate a local promotion clears, versioned and reversible (`app.contribute_federated` / `app.federated_improvement` / `app.adopt_federated`) |
| `60_differential_privacy_memory_training.py` | differential-privacy memory & training — a Rényi/moments `PrivacyAccountant` composes the cumulative `(ε, δ)` a subject's data spends across every memory consolidation and federated contribution (far more tightly than naively summing each step), a per-subject `PrivacyBudget` gates a learning step the way the cost report gates a dollar (refuse, or down-weight by clipping harder), and a per-subject `privacy_report()` makes the spent budget an auditable number — every spend and refusal on the hash-chained audit log (`app.use_privacy_accountant` / `app.set_privacy_budget` / `app.privacy_report`) |
| `61_cross_fleet_reputation_weighting.py` | cross-fleet reputation & weighting — a `ReputationLedger` earns a per-member reliability score from how each federated contribution fared against the no-regression gate (a Beta-Bernoulli posterior over gate outcomes, never raw traffic), the `SecureAggregator` weights a member's contribution by its reputation so a repeatedly-regressing or adversarial member is discounted without being singled out (the discount folded in before the secure-aggregation masks so they still cancel), bounded (`[floor, 1]`) and reversible (adoption still clears the same gate), with every update on the signed audit chain and replayable from it (`app.use_reputation_ledger` / `app.reputation_report` / `ReputationLedger.from_audit`) |
| `62_energy_carbon_accounting.py` | energy & carbon accounting — an opt-in per-run **energy** (watt-hours) and **carbon** (grams CO₂e) estimate, accrued deterministically from the run's token accounting against a per-model (by-tier) intensity and a per-region grid factor, on the *same* cost-report surface (`result.energy_wh` / `result.co2e_grams`, `app.energy_report(...)`) — and budgeted the way the cost report budgets dollars: a run that would exceed its sustainability envelope is refused like a hard cost cap, with the estimate and every refusal on the hash-chained audit log, computed in-process from a deterministic intensity table (`app.use_energy_accounting` / `app.set_energy_budget` / `app.energy_report`) |
